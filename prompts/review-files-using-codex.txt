You are helping me build a RAG-based chatbot for Infosys stock/company research.

  Context:
  - I have already accepted/rejected many NSE PDFs using metadata.
  - The current folder contains 77 markdown files under `temp\markdown-data\review\`.
  - These files were converted from PDFs using Docling.
  - Your task is to decide, for each markdown file, whether it should finally be ACCEPTED or REJECTED for the Infosys RAG knowledge base.
  - This decision must be based on the actual markdown content, not only metadata or filename.

  Important philosophy:
  Reject a document ONLY if there is enough evidence that the entire document is useless for a factual Infosys research chatbot.
  When in doubt, ACCEPT.
  False rejection is worse than false acceptance.

  The chatbot is meant to answer factual questions about Infosys, including:
  - financial results
  - annual/quarterly performance
  - earnings call commentary
  - investor presentations
  - acquisitions
  - partnerships
  - major clients/deals
  - leadership/board changes
  - dividends/buybacks
  - litigation/regulatory/tax matters
  - ESG/BRSR/sustainability
  - business strategy
  - risk factors
  - shareholder meetings, but only if there is substantive discussion/transcript/important resolutions
  - company announcements with long-term factual value

  Reject documents that are only:
  - trading window closure
  - loss of share certificates

  - newspaper publication copies
  - scrutinizer reports/voting results only
  - purely procedural AGM/postal ballot notices with no useful business content
  - duplicate administrative filings
  - depository/compliance certificates
  - investor meeting schedules without presentation/transcript/content
  - clean auditor UDIN reports with no qualification/adverse remark/emphasis matter
  - documents containing only boilerplate legal/procedural text

       revenue, margin, profit, guidance, dividend, buyback, acquisition, collaboration, agreement, litigation, GST, tax, regulatory, board, director, CEO, CFO, transcript,
  presentation, annual report, BRSR, ESG, risk, strategy, financial statements, auditor, qualification, emphasis, adverse, resignation, settlement, cyber, McCamish.

  * decision
  * document_families

  Do **NOT** first analyze all files and then generate the outputs.

  1. Pick one markdown file.

  * Skip files that have already been processed.
  * Continue only with the remaining markdown files.
